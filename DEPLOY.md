# Deployment guide — GitHub Pages & Netlify

This site is static (HTML/CSS/JS). It works when served from a subpath (e.g. `https://<user>.github.io/<repo>/`) because all asset and link paths are **relative** (no leading `/`).

## 2) Checklist — must be true for static hosting

- [x] **Single entry:** `index.html` at the directory you publish (repo root or `/docs`).
- [x] **No absolute paths:** No `href="/..."` or `src="/..."` in HTML; no `url(/...)` in CSS. All paths are relative so they work under a subpath.
- [x] **Purely static:** No server/backend; no Node at runtime. Only HTML, CSS, JS, images, PDF.
- [x] **Build before deploy:** Run `node utils/build.js` before pushing so `index.html`, `about.html`, `experience.html`, `projects.html` exist at root.
- [x] **`.nojekyll`** at root so GitHub Pages doesn’t run Jekyll (avoids underscore-folder quirks).

## 3) Exact deploy steps

### GitHub Pages (project site: `https://<user>.github.io/<repo>/`)

1. **Build and commit the site:**
   ```bash
   node utils/build.js
   git add index.html about.html experience.html projects.html .nojekyll
   git commit -m "Build for deploy"
   git push origin main
   ```

2. **Settings → Pages:**
   - Repo **Settings** → **Pages** (left sidebar).
   - **Build and deployment** → **Source:** “Deploy from a branch”.
   - **Branch:** `main` (or your default) — **Folder:** **/ (root)**.
   - **Save**.

3. **URL:**  
   `https://<your-username>.github.io/Personal_Portfolio/`  
   (Replace repo name if different.)

---

### Netlify

1. **Build and push** (same as above: run build, commit root HTML + `.nojekyll`, push).

2. **Netlify dashboard:**
   - **Add new site** → **Import an existing project** → connect your Git provider and select this repo.
   - **Build settings:**
     - **Build command:** leave **empty** (no build step required if you commit built files).
     - **Publish directory:** ` .` (root) or `/` (so `index.html` is at the published root).
   - Deploy.

3. **Subpath publish (e.g. “Publish subdirectory”):**  
   If you ever publish a subfolder (e.g. `dist`), set **Publish directory** to that folder. This repo is set up for **root** publish, so use **`.`** or `/`.

4. **URL:**  
   `https://<your-site-name>.netlify.app` (or your custom domain).

## 4) How to verify

| Check | URL to try | Success looks like |
|-------|------------|--------------------|
| Home | `https://<user>.github.io/<repo>/` or `.../index.html` | Home page with nav, content, and styles. |
| About | `.../about.html` | About page; no 404. |
| Experience | `.../experience.html` | Experience page; cards and panels work. |
| Projects | `.../projects.html` | Projects page; images and open-link icon load. |
| Assets | Open DevTools → Network; reload | No 404s for `styles/main.css`, `js/app.js`, or `img/...`. |
| Resume | Click “Resume” | PDF opens or downloads. |

If the site was broken before, the usual causes were: (1) no `index.html` at published root (build not run or not committed), or (2) absolute paths like `/img/...` (fixed: build outputs relative paths only). `.nojekyll` avoids Jekyll treating the site as a Jekyll app.

---

## After editing content or components

1. Run **`node utils/build.js`** to regenerate root HTML.
2. Commit the updated `index.html`, `about.html`, `experience.html`, `projects.html`.
3. Push; GitHub Pages / Netlify will redeploy automatically if connected to the same branch.
